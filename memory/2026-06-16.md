## SkillOpt × Deeptechnic 整合（2026-06-16 下午）

### 目标
将 Microsoft SkillOpt（文本空间 skill 优化器）与 Deeptechnic 技术研判系统整合，实现工作流程规则的自动优化。

### 完成的工作

**环境搭建**
- Python 3.12 安装（MacOS .pkg）
- pip install skillopt==0.1.0 成功
- 示例程序跑通（training 输出正常）
- 克隆 GitHub 仓库获取配置模板

**Benchmark 构建**
- benchmark 名称：`deep_tech_dd`
- 评估维度：10项质量检查清单（对 02-技术尽调工作流程规则.md 做 indicator-based scoring）
- 历史数据：4个项目（脉冲红外=1.0, Micro-OLED=0.75, 低空巡航=0.65, 普瑞达=0.55）
- 自定义 evaluator 基于指标关键词匹配（正则）

**首次训练结果**
- 全管道跑通：rollout → reflect → aggregate → select → update → gate
- Deepseek API 作为 optimizer 正常工作，提出 3 条编辑建议
- 但被验证门控拒绝（当前 v3.1 规则在已有指标上已得满分）
- 限制：数据量太小（3 train / 1 val / 1 test），指标基于关键词匹配区分度不够

### 已知限制与下一步方向
- **数据不足**：4条训练数据无法体现优化空间，需要更多项目报告和用户反馈
- **Evaluator 粒度不够**：关键词匹配 vs LLM-as-Judge 的精度差距
- **Rewrite mode 未测试**：当前用 patch mode（增量补丁），rewrite mode（完整重写）可能更适合中文长文档
- 下次进入此对话需重建历史项目数据，建议先扩大数据集再跑训练
