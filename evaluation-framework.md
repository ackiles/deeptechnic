# evaluation-framework

## 一、新技术新市场项目（>=30%）

### 1. 技术维度
- **先进性**：
  - 关键技术指标国内前3或显著超越现有方案
  - 技术路线独特（多路径创新）
  - 无“伪创新”（原理清晰）
- **可行性**：
  - 具备工程化能力（可量产）
  - 成本竞争力（理想：指标优+成本低；可接受：技术优势覆盖迁移成本）
- **适配性**：
  - 与新需求场景高度匹配（契合痛点）
  - 技术对场景不可替代

### 2. 市场维度
- **规模**：
  - 达IPO标准（营收潜力≥行业50%份额）或被并购潜力（稳定营收）
  - 真实需求（交叉验证渗透率，避免BP夸大）
- **需求特点**：
  - 未被充分满足
  - 团队具备市场教育能力（KOL/资本影响力）
- **竞争格局**：
  - 优先有生态资源的细分赛道
  - 规避红海领域（芯片设计、纯SaaS）

### 3. 团队维度
- **核心能力**：
  - 持续技术演进能力（非临时转型）
  - 补充商业化人才（尤其高校团队）
- **创业意愿**：
  - 自筹资金启动、全职投入
  - 规避科研经费补充型（动力不足）

### 4. 商业模式维度
- **商业闭环**：
  - 有早期订单或头部客户POC验证
  - 优先级硬结合（纯SaaS国内难推行）
- **现金流**：
  - 优先自造血
  - 需融资则证明持续性（豪华背景/政府资源）

### 5. 战略与政策
- 聚焦国家主战场（AI、低空经济等）
- 具备战略价值（如ChatGPT级创新）

### 6. 风险规避
- 警惕技术被大公司覆盖、立项周期短
- 避免“伪需求”（需客户实际买单）

## 二、存量替代场景项目

### 1. 技术维度
- **先进性**：关键技术指标显著超越现有；技术路线独特
- **可行性**：可量产；成本降低30%+或性能翻倍，覆盖客户迁移成本（收益-迁移成本=现有收益）
- **适配性**：与存量场景匹配；替代路径清晰

### 2. 市场维度
- **规模**：达IPO或被并购标准；真实需求（交叉验证）
- **需求特点**：效率提升显著；迁移成本可控
- **竞争格局**：规避红海；有生态资源壁垒

### 3. 团队维度
- **核心能力**：持续技术演进；绑定头部客户资源（避免“技术好无人用”）
- **创业意愿**：自筹资金、全职投入

### 4. 商业模式维度
- **商业闭环**：早期订单/POC验证；软硬结合优先
- **现金流**：自造血；需融资则证明能力

### 5. 战略与政策
- 符合政策支持领域
- 经济回报明确（营收增长成熟）

### 6. 风险规避
- 验证技术护城河（防大公司复制）
- 避免迁移成本陷阱（收益>成本+阻力）

## 三、需规避的项目类型

### 1. 技术层面
- 伪创新（原理不清/擦边球宣传）
- 实验室阶段无法量产
- 被大公司技术覆盖

### 2. 市场层面
- 小众市场（未达IPO/被并购标准）
- 红海领域（芯片设计、纯SaaS）
- 伪需求（无实际买单）

### 3. 团队层面
- 临时转型（无持续技术能力）
- 高校团队缺商业化人才（教授任CEO需判断：有创业经历和产业化思维则不是问题；纯学术背景无商业经验则高风险）
- 科研经费补充型（动力不足）

### 4. 商业模式
- 纯软件（SaaS国内难推广）
- 无商业闭环（无订单/POC）
- 过度依赖补贴（无自造血）

### 5. 战略风险
- 非政策支持领域
- 卡脖子技术但资金不足（需大量资金+长周期）
- 偏材料的低效替代项目
