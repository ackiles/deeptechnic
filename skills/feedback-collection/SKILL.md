# Feedback Collection — 零操作的自动反馈采集

## Overview

**你什么都不用做。** 不用填表、不用打分、不用标注。每一次对话——你提的问题、要的修改、说的评价——都会被 signal-classifier 自动识别为质量信号，本 skill 在对话结束时自动汇总成结构化数据，追加到 SkillOpt 训练集。

## When to Use

不需要手动触发。**嵌入在每次对话轮次处理的最后一步：**

```
对话开始 → （执行尽调流程）→ 用户反馈（隐式）→ signal-classifier 实时分类
→ 对话结束/版本切换 → 本 skill 自动触发 → 汇总 → 写入
```

## Process（全自动执行，无需人工介入）

### Step 0: 读取 signal-classifier 的累计信号

在对话结束或版本切换时，自动读取 signal-classifier 在本轮对话中累计的所有信号数据。

### Step 1: 从累计信号生成 10 维评分

将累计的信号按维度聚合，计算最终评分：

| 信号类型 | 对评分的影响 |
|----------|------------|
| 无信号（未提及的维度） | 标记为 null，不参与评分 |
| 仅有正向信号 | 基础分 0.85 + 正向信号 × 加权 |
| 仅有负向信号 | 基础分 0.50 - 负向信号 × 加权 |
| 正负向都有 | 从 0.70 开始，正负向累积调整 |
| 有对比信号（"比上次差"） | 整体 -0.10 偏移 |

### Step 2: 写入项目上下文文件

自动写入 `用户反馈与自评记录.md`——不冗余展示，追加即可。

### Step 3: 追加 SkillOpt 训练数据

自动分集追加到 `skillopt-integration/data/deep_tech_dd/`：

| 场景 | 分集 | 原因 |
|------|------|------|
| 仅有正向信号且用户要求结束 | train | 正样本——满意 |
| 有负向信号，用户要求修改后满意 | val ← v1（负样本）, train ← v2（正样本）| 成对信号 |
| 仅有负向信号，未修改 | test | 负样本——需进一步验证 |
| 无有效信号 | 不追加 | 数据不足 |

### Step 4: 检查触发重训练条件

自动检查 `data/deep_tech_dd/` 下的总数据条数。累计 ≥ 20 条新数据后标记通知。

## 与 signal-classifier 的分工

```
signal-classifier（子 skill）
  职责：每轮对话中实时分类用户消息为维度信号
  触发：每条用户消息
  输出：维度 × 信号方向 × 信号强度的累计数据

feedback-collection（本 skill）
  职责：在对话/版本结束时，汇总信号、生成评分、写入训练数据
  触发：对话结束 / 版本切换
  输出：自评记录.md + 训练数据
```

## Verification（自动检查）

- [ ] signal-classifier 已累计至少 3 条信号 → 必须汇总
- [ ] signal-classifier 没有信号 → 跳过本次，不写入
- [ ] 训练数据已追加（如果≥1条有效样本）
- [ ] 累计数据量已检查，≥20 条则通知触发重训练
