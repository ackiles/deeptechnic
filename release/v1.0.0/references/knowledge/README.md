# References — 参考知识库

> 知识库是使用技术研判系统过程中积累的各类参考材料，按来源分类：
> - 人工整理的产业报告/行业案例
> - 开源 skill（从 agent-skills 等外部来源适配）
> - 项目尽调过程中产生的优质内容（BP 中的好做法、对话中的好思路）
> - 跨项目复用的分析模板和方法论

## 目录结构

```
references/
├── README.md               # 本文件 — 知识库使用说明
├── physics-formulas.md      # 常见技术方向的物理公式库
├── supply-chain-framework.md # 供应商尽调通用框架
├── team-assessment-template.md # 团队评估模板
└── competitor-comparison.md  # 竞品定量对比框架
```

## 使用方式

1. 每次项目尽调启动时，根据需要加载对应的参考文件
2. 尽调过程中发现好的分析思路、BP 中的亮点、用户的反馈建议，可追加到对应文件
3. 跨项目积累后，知识库成为 skill 的支撑材料

## 来源规则

| 来源 | 可录入内容 | 收录标准 |
|------|-----------|---------|
| 产业报告 | 市场规模、竞争格局、技术趋势 | 有明确出处和日期 |
| 行业案例 | 同类项目的历史经验、成功/失败原因分析 | 有具体案例说明 |
| 开源 skill | 从 agent-skills 等仓库适配的技能 | 需经过评审判定适配 |
| BP 中的好内容 | 团队的实际技术路线说明、BOM 分析、竞品调研 | 需脱敏且标注来源 |
| 对话中的好思路 | 用户在反馈中提出的好建议、我生成的优质分析 | 需凝练为通用规则 |
