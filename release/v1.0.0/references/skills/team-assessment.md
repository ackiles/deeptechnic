# Team Assessment — 团队能力评估

## Overview

"团队有多年相关经验"是 BP 中最常用的描述之一，也是最难验证的。本 skill 的任务是把这句话拆解为可验证的逐人分析——谁、做过什么、和这个项目直接相关吗、全时投入吗。

关键原则：**一份只有"强""经验丰富"等定性评价的团队评估 = 没有评估。**

## When to Use

- 第一轮初评 — 拿到核心团队名单后
- 团队回复后 — 团队补充了新成员或新履历信息
- 发现履历疑点 — 公开信息与 BP 描述不一致

## The Process

### Step 1: 列人

从 BP 和公开信源（LinkedIn、天眼查、学术主页、ORCID）收集所有核心成员信息，按以下格式逐人列出：

```
姓名：[中文名/英文名]
Title：[CEO/CTO/首席科学家...]
教育背景：[学校/专业/学位]
过往经历：[公司/岗位/年限/具体项目]
全时投入：[是/否]
可查证来源：[LinkedIn链接/ORCID/Google Scholar/...]
```

逐人到这个粒度，不做"核心团队整体评价"的笼统描述。

### Step 2: 逐人匹配

对每个人回答两个问题：

1. **过往经历是否与此项目核心技术直接相关？**
   - 相关：此前做过完全相同或高度重叠的技术方向
   - 部分相关：此前做过同一大领域但不同子方向（如做过可见光CMOS但没做过红外焦平面）
   - 不相关：此前做的事情和当前项目没有技术交集

2. **全时投入还是兼职？是否需要依赖此人才能推进项目？**
   - 关键技术依赖兼职人员 → 高风险信号
   - CEO 是兼职教授 → 评估是否已全职、是否转让了IP

每个判断附带证据链（引用来源），非主观推测。

### Step 3: 识别结构性缺口

回答核心问题：**团队在哪个关键技术环节上存在能力空白？**

- 做红外硬件但没有一人有 ROIC/斯特林制冷机/混合集成经验？
- 做AI芯片但没有一人有 ASIC 流片经验？
- 做生物传感器但没有一人有电化学测试经验？

缺口的判断标准：这个环节可不可以外包？如果外包，IP归属、优先级冲突、交付周期风险各有多大？

### Step 4: 竞品团队对比

- 直接竞品公司的核心团队成员配置如何？
- 我们的团队在哪些关键岗位上有/没有对位的人？
- 如果缺少，是可以通过招聘补齐的（在合理周期内），还是行业稀缺人才？

### Step 5: 综合评价

输出团队评估结论：
- 团队概况：N 人全职 / M 人兼职，覆盖了 / 未覆盖的关键岗位
- 最大缺口：XX 方向 N/A
- 外部协作：依赖 XXX 公司/机构，风险评估
- 总体判断：绿（结构完整）/ 黄（有缺口可补）/ 红（结构性缺陷）

## Rationalizations

| BP话术 | 反驳 |
|--------|------|
| "团队经验丰富" | "请逐人列出姓名、单位、具体项目，我们将独立核实" |
| "核心成员来自XX大厂" | "在大厂任职与能独立创业是两回事，请给出该成员在XX大厂的实际产出" |
| "我们有技术顾问团队" | "顾问是兼职，每周几小时？核心开发谁做？" |
| "团队共事多年" | "共事多年是好事，但有没有一起从0到1做出一个产品的经历？" |

## Red Flags

1. 团队成员的领英/学术主页/论文信息在公开渠道**完全找不到**
2. 核心成员**全部**非全职（关键研发依赖兼职）
3. CEO 为教授且未全职投入、知识产权未转让
4. 关键技术只有一个成员会，此人兼职或随时可能离开
5. BP中的团队介绍全是"前XX大厂"没有具体项目

## Verification

- [ ] 逐人列出姓名/title/教育背景/过往经历/全时状态
- [ ] 逐人给出匹配度判断（相关/部分相关/不相关）
- [ ] 识别了至少 1 个结构性能力缺口（或无缺口的证据）
- [ ] 评估了外部协作的风险
- [ ] 与核心竞品做了团队配置对比
