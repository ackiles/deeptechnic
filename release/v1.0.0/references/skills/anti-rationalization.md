# Anti-Rationalization — BP 话术反驳框架

## Overview

技术尽调中，团队/BP 中的很多话术听起来合理，背后是常见的搪塞逻辑。这个 Skill 提供一套系统化的反驳框架，帮助分析师（Deeptechnic）快速识别这些话术并给出标准追问。

本 Skill 训练数据的积累是 SkillOpt 自动优化工作流程规则的核心数据源——每次命中一条话术，就是一个有价值训练样本。

## When to Use

- **阅读 BP 的过程中** — 边读边识别话术，标注在 Claim 验证表中
- **回答团队书面反馈时** — 团队避实就虚时自动引用对应反驳
- **生成访谈提纲时** — 将命中话术转化为面试追问

## The Process

### Step 1: 话术扫描

以怀疑的态度阅读 BP 中所有"我们是XX"、"我们的指标达到XX"、"我们比竞品好XX"的表述。特别关注：

1. **绝对性措辞** — 「唯一」「首创」「独家」「颠覆性」「革命性」
2. **模糊量化** — 「很多客户」「行业领先」「显著提升」「大幅降低成本」
3. **避实就虚** — 「这个参数还在优化中」「我们关注的是整体方案而非单个指标」
4. **不完整对比** — 只比好了、不比差了；只比参数、不比成熟度
5. **过度承诺** — 同时满足物理上难以同时成立的多个约束
6. **指代不明** — 「我们通过独家工艺实现了…」但不说是哪个工艺、「我们有核心算法」但不展开

### Step 2: 匹配反驳库

从下表中找到匹配的反驳，将命中话术和对应的标准反驳记录到评估文档中。

### Step 3: 转化为追问

每条匹配的话术转化为 P0/P1/P2 优先级追问：

- P0（必须回答，否则项目不可推进）— 涉及技术可行性根基的
- P1（建议回答）— 涉及竞争壁垒或商业化可行性的
- P2（可暂缓）— 涉及市场前景或发展计划的

## Rationalization 数据库

### 技术类

| Level | BP 话术 | 标准反驳 | 追问转化 |
|-------|---------|---------|---------|
| 🔴 | "我们的指标达到了国际领先水平" | 请提供第三方测试报告和量化对比数据。无第三方数据 = 不可验证的自述 | P0：提供第三方检测报告 + 至少 2 个竞品在同样条件下的对比数据 |
| 🔴 | "这个参数还在优化中" | 没有初步数据 = 无法评估基础可行性。"优化"的前提是有基线数据 | P0：当前最优值是多少？测试条件是什么？ |
| 🔴 | "我们的技术路径是独创的，没有竞品" | 没有竞品 = 要么没有市场，要么不可行，要么团队没做功课 | P0：请列出全球范围内所有走相似路线的团队/公司，说明差异 |
| 🟡 | "我们的算法比所有竞品都好" | 请给出具体竞品型号、参数、同条件测试数据 | P1：用同数据集/同场景做 A/B 对比测试 |
| 🟡 | "我们的技术是跨时代的、颠覆性的" | 颠覆了哪个路线？替代成本？客户迁移的理由？ | P1：具体来说，替代现有方案需要客户付出什么成本？ |
| 🟡 | "我们只展示部分数据，核心参数保密" | 对投资人最大限度保密 = 没有或不想被验证 | P1：是否需要签 NDA？签署后可以展示全部测试数据？ |

### 竞争类

| Level | BP 话术 | 标准反驳 | 追问转化 |
|-------|---------|---------|---------|
| 🔴 | "我们是市场上唯一的XX" | 用公开信息验证。BP说一个人时，查出来至少3-5家才算对 | P0：你如何看待以下 N 家竞品？（给出独立搜索到的竞品名单） |
| 🔴 | "我们的竞品只有FLIR/国外巨头" | 主动搜索国内中小竞品。国产替代已完成，国内竞争者比国外更可怕 | P0：国内以下供应商的产品是否构成直接竞争？（列出国内竞品） |
| 🟡 | "我们对标的竞品价格是XX万" | 对标价 ≠ 实际成交价，且新产品初期良率低、成本高于对标 | P1：请提供竞品的实际成交价或渠道报价，而非官网标价 |

### 团队类

| Level | BP 话术 | 标准反驳 | 追问转化 |
|-------|---------|---------|---------|
| 🔴 | "我们的核心成员来自XX大厂" | 在大厂做过 ≠ 能独当一面。请给出：谁、什么部门、什么职位、做什么具体项目 | P0：列出每位核心技术成员的全名+教育背景+在XX大厂的具体项目和角色 |
| 🔴 | "我们的技术顾问是全明星阵容" | 顾问 = 兼职投入，参与度有限。核心开发是谁？ | P0：顾问的投入比例和时间？核心全职团队中有多少人有实战经验？ |
| 🟡 | "核心团队共事多年，默契很好" | 共事多年 = maybe 养老思维，未必适合创业 | P1：过去有没有一起从0到1做成一个产品/公司的经历？ |
| 🟡 | "我们的创始人XX是教授/博导" | 教授=工作时间有限制、可能与教职利益冲突 | P1：是否全职投入？知识产权是否已转让给公司？是否与教职冲突？ |

### 市场/商业类

| Level | BP 话术 | 标准反驳 | 追问转化 |
|-------|---------|---------|---------|
| 🟡 | "目标市场规模有XX亿" | 请给出计算方法和数据来源；二手引用要追溯到原始报告 | P2：请补充市场规模的计算假设和数据来源 |
| 🟡 | "我们的成本可以降到现在的1/10" | 请提供 BOM 拆解或供应商正式报价 | P1：目前的 BOM 成本是多少？降本路径和时间表是什么？ |
| 🟡 | "已经有多个客户在洽谈" | 客户名称？合作阶段？是否已签合同/打样/测试？ | P1：请列出所有正在接触的客户及合作阶段 |

## Red Flags

1. **BP 中没有一张实测数据截图/测试报告** — 所有数据都是文字描述
2. **拒绝在签 TS 前提供任何核心数据** — "签了投资意向书再给" = 数据大概率有问题
3. **把未来目标当现在成就说** — "预计达到XXX""目标实现XXX"与"已实现XXX"混用
4. **对任何技术问题都能马上给出长篇解释** — 有备而来 ≠ 技术扎实，可能是精心准备的标准答复
5. **团队全部非全职** — 绝大多数成功的硬科技创业项目至少有一位全职技术带头人
