# Signal Classifier — 隐式反馈信号分类器

## Overview

用户不会主动说"这个维度我给你打0.85分"。但用户说的每一句话——"再分析一下XX"、"这个写得好"、"和上次比差一些"——都隐含着对报告质量维度的评价。

本 skill 的职责：**每一次对话轮次中，自动分类用户的消息，提取满意度信号，无需任何额外操作。**

## Core Logic

用户的下一条消息就是最佳反馈时机。不需要在对话结束时才采集，而是 **对每一条用户消息立即做信号分类**。

### 信号映射规则

| 用户行为/话语模式 | 映射的维度信号 | 评分影响 |
|------------------|--------------|---------|
| "再分析一下XX方面" / "XX这部分不够" | 对应维度 **负向** | -0.1 ~ -0.2 |
| "这个写得好" / "到位" / "透彻" | 综合 **正向** | +0.05 ~ +0.1 |
| "和BP说的不一样" / "不是这样的" | `independent_analysis` **正向** | +0.1 |
| "你错了" / "这个不对" | `claim_validation` **负向** | -0.2 |
| "和上次比差一些" | 综合 **负向对比** | -0.1 |
| "再加XX分析" | 对应维度 **缺漏** | -0.15 |
| "按刚才讨论的修改" | 隐式：当前版本总体上不够好 | 各维度 -0.05 |
| 未提及的维度 | 无信号 | 保持 null |
| "先这样吧" / "今天就到这" | 对话结束信号，触发最终汇总 | 汇总所有累计信号 |
| "做得不错" / "可以了" | 综合 **正向确认** | +0.1 |
| （对某段分析没有反应） | 中立，但分送继续推进 | 保持 |
| （提出要求后看更新版）"好" / "行" | 更新版 **正向确认** | +0.15 |

### 信号累积规则

同一个维度多次信号的累计方式：

```
第1次：用户说"竞品分析不够" → independent_competitor: -0.15
第2次：用户说"你再看看XXX这家公司" → independent_competitor: -0.20（累计）
第3次：更新后用户说"这次竞品分析可以了" → independent_competitor: +0.15（修正）

最终：independent_competitor = 0.5 - 0.15 - 0.20 + 0.15 = 0.30
```

### 何时触发汇总写入

在以下任一事件触发时，将累计信号汇总为训练数据：

1. **用户明确结束信号** — "先这样吧"、"今天就到这"、"下次再聊"
2. **跨版本交付** — 从 v1 到 v2 之间的所有信号 → 统一汇总为 v1 的评价
3. **项目对话关闭** — 会话结束前的最后一条消息

## Process

### 无需主动执行的 Process

本 skill 不需要显式的 Process 步骤。其逻辑嵌入在每一次消息处理中：

```
每条用户消息 → 自动分类（0延迟）→ 累计到对应维度
对话结束 → 自动汇总 → 写入文件 → 追加训练数据
```

### 汇总输出格式

```
## 自动反馈汇总
触发时间：[对话结束/版本切换]
累计信号数：N 条
覆盖维度：N 个

| 维度 | 累计信号 | 最终评分 | 证据链 |
|------|---------|---------|--------|
| claim_validation | 正向2条,负向1条 | 0.85 | "这个验证做得好" |
| per_person_assessment | 负向2条 | 0.30 | "再看一下XX的背景" |
| independent_competitor | 无信号 | null | — |
```

## Integration

本 skill 通过以下方式嵌入：

1. 在 `iteration-merge` 的 Step 0 中引用：回溯用户问题前先做信号分类
2. 在 `feedback-collection` 的 Step 0 中引用：汇总前先读取累计信号
3. 在 `02-workflow-rules.md` 的系统原则第7条中引用：每次对话轮次自动执行
